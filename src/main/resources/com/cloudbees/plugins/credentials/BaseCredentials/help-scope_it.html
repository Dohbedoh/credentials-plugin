Quest'opzione determina dove può essere utilizzata questa credenziale.
Sistema
Questa credenziale è disponibile solo per l'oggetto a cui essa è stata associata. Tipicamente si utilizzano le credenziali con scope di sistema per l'autenticazione della posta elettronica, la connessione agli agenti, ecc., ossia per i casi in cui l'istanza stessa di Jenkins le utilizza. A differenza dello scope globale, ciò restringe in modo significativo gli ambiti in cui può essere utilizzata la credenziale, il che le fornisce un grado di confidenzialità più elevato.
Globale
Questa credenziale è disponibile per l'oggetto a cui la credenziale è associata e per tutti gli oggetti figli. Tipicamente si utilizzano le credenziali con scope globale per gli oggetti richiesti dai processi.

In generale, una credenziale è definita in un posto (ad es. la pagina di configurazione delle credenziali nella sezione "Configura Jenkins") e utilizzata in un altro (ad es. durante la connessione a un nuovo agente tramite SSH). Lo scope consente di affermare "questa credenziale è utilizzata solo in questi posti" analizzando la relazione fra le due posizioni.
